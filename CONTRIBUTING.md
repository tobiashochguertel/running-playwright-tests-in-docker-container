# Contributing Guidelines for Examples

Thank you for your interest in contributing to the Running Playwright Tests in Docker Container repository! We welcome contributions from everyone. Here are some guidelines to help you get started.

## Guidelines for Contributing New Examples

1. **Fork the Repository**: Start by forking the repository to your own GitHub account.
   
2. **Clone Your Fork**: Once forked, clone your repository to your local machine using:
   
   ```bash
   git clone https://github.com/your-username/running-playwright-tests-in-docker-container.git
   ```

3. **Create a New Branch**: Before making changes, create a new branch specific to your example:
   
   ```bash
   git checkout -b example-your-example-name
   ```

4. **Add Your Example**: Add your example to the appropriate directory within the repository. Ensure that it is organized and well-documented.

5. **Test Your Example**: Make sure to test your example thoroughly to verify that it works as intended.

6. **Commit Your Changes**: After you have made your changes, commit them with a clear message:
   
   ```bash
   git commit -m "Add example for your-example-name"
   ```

7. **Push Your Changes**: Push your changes to your forked repository:
   
   ```bash
   git push origin example-your-example-name
   ```

8. **Open a Pull Request**: Finally, open a pull request in the original repository. Be sure to describe what your example does and why itâ€™s useful.
